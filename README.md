# DQuad - Windows 四象限文件管理器

一款轻量级 Windows 文件管理器，采用四象限布局，支持多标签页和书签管理。

**当前版本：0.2.0**

## 功能特性

- **四象限布局**：同时浏览四个目录，提高工作效率
- **多标签页**：每个象限独立标签页，支持新建/关闭/切换/拖拽排序
- **书签管理**：支持分组、拖拽排序、导入/导出
- **面包屑导航**：快速跳转到任意父目录
- **文件操作**：复制、剪切、粘贴、删除、重命名
- **视图切换**：支持详细信息/大图标/小图标/列表/平铺五种视图模式
- **环境变量设置快捷**：工具栏新增按钮，一键打开系统环境变量配置界面
- **文件拖拽**：支持跨象限文件拖拽操作
- **自定义命令**：可配置快捷命令菜单，支持占位符替换（示例包含环境变量命令）
- **系统集成**：右键菜单支持系统原生操作
- **状态持久化**：自动保存窗口布局、标签页状态和视图设置

## 编译指南

本项目推荐使用 **MinGW-w64 (64-bit)** 环境进行编译。由于程序包含嵌入式字体和图标资源，建议优先使用自动化脚本。

### 方法 A：使用自动化脚本 (推荐)

项目根目录下提供了基于 PowerShell 的一键编译脚本，它会自动处理进程占用清理、资源文件编译及主程序链接。

在终端中执行：

```powershell
.\scripts\build.ps1

```

**脚本功能：**

* **进程清理**：自动检测并强制结束正在运行的 `DQuad.exe`，避免文件锁定。
* **资源预检**：自动调用 `windres` 编译 `.rc` 资源（含图标和字体）。
* **自动构建**：使用优化参数（`-O2 -s`）生成体积精简的 64 位 GUI 程序。

---

### 方法 B：手动命令行编译

如果你希望手动执行编译过程，请按照以下步骤操作：

#### 1. 编译资源文件

将图标和字体打包为 Windows 资源对象：

```bash
windres src/DQuad.rc -o src/DQuad_res.o

```

#### 2. 编译并链接主程序

使用 GCC 进行最终的链接。注意需要链接 Windows 核心库以支持通用控件、OLE 对象的渲染：

```bash
gcc src/main.c src/DQuad_res.o -o DQuad.exe -std=c11 -O2 -s -mwindows -lcomctl32 -lole32 -luuid -lshlwapi -DUNICODE -D_UNICODE

```

**核心编译参数说明：**

* `-O2 -s`: 启用二级优化并去除符号表，显著减小可执行文件体积。
* `-mwindows`: 纯 GUI 模式，启动时不弹出控制台黑窗口。
* `-lcomctl32 -lshlwapi`: 链接 Windows 现代 UI 控件和 Shell API 库。
* `-DUNICODE`: 开启宽字符支持，确保程序能正确处理中文路径和 Emoji 图标。

---

**注意事项：**

* **环境要求**：请确保 `gcc` 版本为 15.1.0 或更高（x86_64-ucrt 架构）。
* **资源缺失**：若 `DQuadLOGO.ico` 不存在，脚本将回退至无图标编译模式。

## 配置文件

- `bookmarks.ini` - 书签配置
- `state.ini` - 窗口状态
- `commands.ini` - 自定义命令配置

## 自定义命令（commands.ini）

点击工具栏终端按钮会弹出命令菜单，命令按 `commands.ini` 配置执行。  
菜单也提供：`文件 -> 打开命令配置文件`。

支持占位符：

- `{cwd}` 当前标签目录
- `{sel}` 当前列表首个选中项完整路径（无选中时为空）
- `{name}` `{sel}` 的名称（无选中时为空）
